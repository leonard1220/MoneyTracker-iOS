# 🧪 FinFlow 测试指南 (Testing Guide)

本文档旨在帮助您验收最新开发的 v0.3+ 功能。请按照以下步骤进行测试，确保所有新特性均工作正常。

## 📥 获取最新版本
1.  确保您已将最新代码 Push 到 GitHub。
2.  前往 GitHub 仓库的 **Actions** 标签页。
3.  等待最新的 Workflow (Build) 完成（通常显示为绿色 ✅）。
4.  下载构建产物 (Artifacts) 中的 `.ipa` 文件。
5.  使用 **Sideloadly** 将 `.ipa` 安装到您的 iPhone 上。

---

## ✅ 验收清单 (Checklist)

### 1. 🚀 全新引导页 (Onboarding Flow)
> **测试前提**：首次安装或卸载重装 App。
- [ ] **启动 App**：是否看到了新的欢迎界面？
- [ ] **语言选择**：选择 "中文" 或 "English"，确认界面语言是否变化。
- [ ] **夜间模式**：确认 App 是否默认呈现深色模式 (Dark Mode)。
- [ ] **初始设置**：按照流程创建第一个现金账户（例如：余额 5000）。
- [ ] **完成**：点击“开始使用”，是否顺利进入首页？

### 2. 🏠 首页改版 (Dashboard Redesign)
- [ ] **总资产卡片**：是否看到了带有光泽感的渐变卡片？数字滚动动效是否流畅？
- [ ] **快捷操作**：
    - [ ] 点击 **"记一笔" (Record Transaction)**：是否弹出了添加交易的页面？
    - [ ] 点击 **"订阅" (Subscriptions)**：是否进入了订阅管理页面？
- [ ] **账户列表**：在快捷操作下方，是否有一个可以左右滑动的账户列表？
    - [ ] 尝试横向滑动。
    - [ ] 点击最右侧的 **"+"** 号，是否进入添加账户页面？
- [ ] **净资产趋势图**：在账户列表下方，是否看到了一条曲线图？（*注：首次运行可能只有一个点，需要多天数据才能连成线*）。

### 3. 📅 订阅管理 (Subscription Management)
- [ ] **进入入口**：点击首页的 "订阅" 对话框。
- [ ] **新增订阅**：点击右上角 "+"。
    - [ ] 输入名称（如 Netflix）、金额（如 30）、周期（每月）。
    - [ ] 选择图标和颜色。
    - [ ] 保存。
- [ ] **查看列表**：
    - [ ] 列表顶部是否显示了 **"每月预估支出"** 总额？
    - [ ] 刚才添加的订阅是否在列表中？
    - [ ] 显示的下次扣款日期是否正确？

### 4. 🎭 情绪消费分析 (Emotional Spending Analysis)
- [ ] **录入数据**：
    - [ ] 回到首页，点击“记一笔”。
    - [ ] 记一笔支出，**务必选择一个心情**（如：开心 Happy）。
    - [ ] 再记一笔，选择另一个心情（如：生气 Angry）。
- [ ] **查看报表**：
    - [ ] 进入底部的 **"报表" (Reports/Chart)** 标签页。
    - [ ] 找到紫色卡片 **"情绪消费分析"** 并点击。
    - [ ] **洞察卡片**：是否告诉您哪个心情花钱最多？
    - [ ] **图表**：柱状图是否显示了不同心情的支出对比？
    - [ ] **列表**：下方是否有详细的统计数据？

### 5. 📤 数据导出 (Enhanced Export)
- [ ] **进入入口**：点击底部的 **"设置" (Settings)** 标签页 -> **"数据管理"**。
- [ ] **导出 CSV**：点击 "导出交易记录 (CSV)"。
- [ ] **分享**：选择 "保存到文件" 或通过微信/AirDrop 发送给电脑。
- [ ] **验证内容**：
    - [ ] 用 Excel 或 WPS 打开文件。
    - [ ] **关键检查**：中文是否显示正常（没有乱码）？
    - [ ] **新增列**：是否有 **"心情" (Mood)** 这一列？
    - [ ] **备注**：带有逗号或换行的备注是否格式正确？
